# Caching HTTP Proxy

Многопоточный HTTP прокси-сервер с кэшированием GET-запросов.

## Описание

Прокси-сервер кэширует HTTP GET-запросы и отдаёт повторные запросы из кэша. Поддерживает параллельное чтение из кэша несколькими клиентами (streaming cache).

Особенности:
- Многопоточная обработка клиентов (thread pool)
- Кэш с автоматической очисткой устаревших записей (5 минут)
- Streaming cache: клиенты получают данные параллельно с записью в кэш
- Поддержка только GET-запросов, только успешные ответы (200 OK) кэшируются

## Сборка

```bash
make
```

## Использование

```bash
# Запуск на порту 80 (требует root)
sudo ./build/cache_proxy

# Запуск на порту 8080
./build/cache_proxy -p 8080

# Запуск с указанием порта (длинный формат)
./build/cache_proxy --port 8080

# Показать справку
./build/cache_proxy -h
```

### Опции

- `-p, --port <PORT>` - порт для прослушивания (по умолчанию: 80)
- `-h, --help` - показать справку

## Пример

```bash
# Запускаем прокси
./build/cache_proxy -p 8080

# В другом терминале тестируем
curl -x http://localhost:8080 http://example.com
```

## Очистка

```bash
make clean
```
